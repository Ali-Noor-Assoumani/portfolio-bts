Administration Windows Server & PKI

Mise en place d'un contrôleur de domaine Windows Server 2022. Ce projet se concentre sur la résolution de noms (DNS), la création d'une infrastructure à clés publiques (AD CS), et l'intégration de l'authentification centralisée avec d'autres services de l'infrastructure.

Mon cas d'usage : Identité, Résolution et Sécurité

Mon serveur Windows est le pilier identitaire de mon infrastructure. Mes interventions se sont concentrées sur trois axes majeurs :

  • Le rôle DNS : Indispensable pour la résolution de noms au sein de mon réseau local, permettant aux machines de communiquer efficacement.
  • Le rôle AD CS (Autorité de Certification) : Déploiement d'une PKI interne pour générer des certificats numériques, essentiels pour chiffrer les échanges sur le réseau en HTTPS.
  • L'intégration avec osTicket : J'ai relié mon serveur Linux Debian (osTicket) à mon Windows Server pour permettre aux utilisateurs de se connecter au support informatique en utilisant leurs identifiants Active Directory (LDAP).
